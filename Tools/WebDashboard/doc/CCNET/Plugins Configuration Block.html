CruiseControl.NET : Plugins Configuration Block
This page last changed on Jan 08, 2010 by csut017.

The Web Dashboard comes pre-configured to use a default set of plugins. To change what plugins you want to use for your CruiseControl.NET deployment, use the <plugins> section of the Dashboard configuration.

The section is split up into 4 parts, representing the Farm, Server, Project and Build views available in the Dashboard. Each section can be configured with any number of plugins. Most Build Plugins can be configured just to be used for certain Projects.

Example:

<plugins>
    <customTemplates>some_folder</customTemplates>
    <farmPlugins>
    .
    .
    </farmPlugins>
    <serverPlugins>
    .
    .
    </serverPlugins>
    <projectPlugins>
    .
    .
    </projectPlugins>
    <buildPlugins>
    .
    .
    </buildPlugins>
</plugins>

the <customTemplates> section is available from version 1.4.3 onwards. This provides some theming support for the Dashboard.
The folder is relative to the dashboard folder. And must contain all files from the templates folder. The files in this custom templates folder can be
easily modified, they will not be overwritten when you install a new version of CCNet.

The following plugins are shipped with CruiseControl.NET:

Custom Templates

  • Custom templates

Farm Plugins

  • CCTray Download Plugin
  • Dashboard Administration Plugin
  • Farm Report Farm Plugin

Server Plugins

  • Server Information Server Plugin
  • Server Log Server Plugin
  • Server Report Server Plugin
  • ViewServerQueue Server Plugin
  • Project Configuration Server Plugin

Project Plugins

  • Latest Build Report Project Plugin
  • modificationHistoryProjectPlugin
  • Project Report Project Plugin
  • Project Statistics Plugin
  • RSSFeedProjectPlugin
  • Server Log Project Plugin
  • View All Builds Project Plugin
  • View Configuration Project Plugin
  • XSL Report Project Plugin (Remote - CCNET Community)

Build Plugins

  • Build Log Build Plugin
  • Build Report Build Plugin
  • Multiple XSL Report Build Plugin
  • XSL Report Build Plugin

If you are interested in writing your own Dashboard plugins, read Developing Web Dashboard Plugins.
