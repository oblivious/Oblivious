CruiseControl.NET : CCNet 1.0 Final Release Notes
This page last changed on Nov 16, 2005 by orogers.

This is the official 1.0 release for CruiseControl.NET. These release notes show the list of changes since 1.0 RC2. For a list of all of the fixes included in the 1.0 release, please check the jira site.

Release Notes - CruiseControl .NET - Version 1.0 Final

This is a list of Jira Issues resolved by this release :

Bug

  • CCNET-204 - Latest link on new Web Dashboard app should be dynamic
  • CCNET-243 - Latest cvs server breaks date parser for "log" command
  • CCNET-482 - Incorrect parsing of modifications log on newest CVS server
  • CCNET-523 - QuietPeriod can get stuck in an infinite loop
  • CCNET-545 - CCNet should strip illegal characters out of external content before merging it into the build log
  • CCNET-570 - CCTray: display of "next build check" time when no triggers defined
  • CCNET-575 - ClearCase: rmbraches should not trigger a build
  • CCNET-579 - Iteration Labeller does not correctly generate the initial label

New Feature

  • CCNET-331 - Seapine SurroundSCM Integration
  • CCNET-488 - Vault: Add temporary labelling support and get source by label
  • CCNET-515 - Bundle the CCTray msi with the server msi and provide a link to it from the dashboard so developers can download and install it
  • CCNET-521 - Monitoring of ccnet.config file should be configurable
  • CCNET-566 - Create new UrlTrigger (Steve Norman)
  • CCNET-572 - Add BitKeeper support (Harold L Hunt)
  • CCNET-573 - Add DateLabeller for generating labels in yyyy.mm.dd.build format (Andy Johnstone)
  • CCNET-567 - Create stylesheet to transform output from MSTest
  • CCNET-600 - Ant Reporting Support
  • CCNET-602 - Action Filter for FilteredSourceControl

Improvement

  • CCNET-271 - ClearCase: confluence should make clear need to have build machine clock sync'd with ClearCase server
  • CCNET-365 - Clarify use of CVSROOT for CVS plugin
  • CCNET-560 - CCTray should optionally add itself to the Startup menu group during installation
  • CCNET-561 - Create summary stylesheet for NCover to display total test coverage on report page
  • CCNET-562 - Set initial logging level to Debug to help users more easily configure and debug their server
  • CCNET-563 - Should report a better error if the working directory does not exist
  • CCNET-565 - VSS Username and password should be optional (Jeffrey D. Parker)
  • CCNET-569 - CCTray should store its settings file in the user profile directory
  • CCNET-577 - CVS: capture revision numbers from cvs log output
  • CCNET-582 - Add configurable working directory for Vault
  • CCNET-595 - Quiet Period: Improve behaviour when build server clock is out of sync with source control server
  • CCNET-603 - Add BuildType and Project params to Devenv task
  • CCNET-604 - Add support for german localisation for Visual Source Safe
