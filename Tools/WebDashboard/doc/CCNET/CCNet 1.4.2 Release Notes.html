Highlights of the CCNet 1.4.2 release

Interaction with Subversion and other external processes fixed

The 1.4.1 release had a bug that prevented ccnet from using external processes correctly. Because of a race condition, CCNet would not receive all of the output from external processes. The race condition has been fixed in this release.

Multiple Subversion repository support fixed

There was a bug in previous releases that affected projects using multiple subversion repositories. The bug would cause ccnet to use the revision number from one repository in its requests to the other repository. The fix for this bug is included in the 1.4.2 release.

Still contains all the improvement and fixes from 1.4.1!

The 1.4.1 release had 104 improvements, enhances and fixes. Upgrading to 1.4.2 will include all those fixes as well.
