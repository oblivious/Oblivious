CruiseControl.NET : The Server Console Application
This page last changed on Jan 26, 2010 by dhommel.

You can launch the CruiseControl.NET server by executing the ccnet.exe file, which is located in the server folder of the extracted distribution.

Before doing this, make sure to setup the Server Application Config File.

Executing the server using the --help flag (eg. ccnet.exe --help) will display a list of the command-line arguments accepted by the server. They are:

Command Line Long Option Description Default
--config:configfilename The config file to use. This is useful if you have multiple config files, or are launching the ccnet application from a different directory to the location of the config file ccnet.config in the directory you launch CCNet from
--remoting:[on/off] Whether the CCNet Server should open the reporting/management port configured in the Application Config File on
--project:projectname Sometimes its useful to debug a specific project. If you use this option, CruiseControl.NET will start just the specified project, force a build on it, then shut down the server No project specified
--validate Verify that the configuration is valid. Error messages will be reported to the console and to the log file. Run the console
--errorpause:[on/off] Do not pause the server after an error occurs. Pause on error
--logging:[on/off] Do not log any messages - this will override any log4net configuration in ccnet.exe.config. Log messages
--help Display the list of available commands.  

CruiseControl.NET by default opens a server port to listen for external requests. Make sure to use the --remoting:off option if you do not want this behaviour.

For parsing the commandline arguments CruiseControl.NET uses the Mono.Options class.
Mono.Options allows to specifiy a long and a short version for arguments and also different styles of passing them on the commandline.
The following table shows the short option equivalent of the above long options. All of the options also work if started with a slash (/) instead of a dash (-) and the colon (:) can be replaced with the equals sign (=).

Example:
ccnet.exe /logging:off
or
ccnet.exe --logging=off
or just
ccnet.exe /l:off

Command Line Short Option Description Default
-c:configfilename The config file to use. This is useful if you have multiple config files, or are launching the ccnet application from a different directory to the location of the config file ccnet.config in the directory you launch CCNet from
-r:[on/off] Whether the CCNet Server should open the reporting/management port configured in the Application Config File on
-p:projectname Sometimes its useful to debug a specific project. If you use this option, CruiseControl.NET will start just the specified project, force a build on it, then shut down the server No project specified
-v Verify that the configuration is valid. Error messages will be reported to the console and to the log file. Run the console
-e:[on/off] Do not pause the server after an error occurs. Pause on error
-l:[on/off] Do not log any messages - this will override any log4net configuration in ccnet.exe.config. Log messages
-h|-? Display the list of available commands.
