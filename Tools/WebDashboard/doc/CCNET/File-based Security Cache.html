File-based Security Cache

Provides an in-memory cache for sessions that is backed by a store on disk.

This cache will store the sessions details in memory for quick-access. Whenever a session is changed it also writes a copy of the details to a file on the disk. Then when the security manager is restarted it loads all the sessions from disk.

Version

This security item is available from version 1.5.0.

Examples

<cache type="fileBasedCache" duration="10" mode="sliding" location="C:\sessions\"/>

Configuration Elements

Node Description Type Required Default Version
type The type of caching to use - must be fileBasedCache. String Yes n/a 1.5.0
duration The duration, in minutes, that a session is stored for. Integer No 10 1.5.0
mode The type of expiration period to use. Options are either Sliding (the expiry time is moved every time a security request is made) or Fixed (expiry time never changes). String:
  • Sliding
  • Fixed
 No Sliding 1.5.0
