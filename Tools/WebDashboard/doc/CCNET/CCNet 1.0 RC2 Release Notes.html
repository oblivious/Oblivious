CruiseControl.NET : CCNet 1.0 RC2 Release Notes
This page last changed on Nov 14, 2005 by orogers.

This is the second Release Candidate for CruiseControl.NET 1.0. It incorporates a number of bug fixes and enhancements based on the feedback that we have received after releasing the first release candidate. We welcome futher feedback on this candidate to improve the final distribution of 1.0.
Release Notes - CruiseControl .NET - Version 1.0 RC2

  • CCNET-486 - Xml log files contain invalid xml if external applications output invalid xml to the command line
  • CCNET-508 - CCNet should automatically create the artifact directory, if it does not exist.
  • CCNET-509 - Fix null reference and tasks without id for Synergy provider
  • CCNET-516 - Modifiers are not getting added to the recipients email list on failed builds
  • CCNET-517 - MsBuild XmlLogger throws path format not supported exception
  • CCNET-522 - Input redirect fails in batch files launched from CCService.exe
  • CCNET-527 - ThoughtWorks.CruiseControl.Core.CruiseControlException: Unable to parse vss date:

  • CCNET-535 - MsBuildTask shows success even after a failed build
  • CCNET-536 - MsBuildTask should support multiple arguments in the buildArgs parameter
  • CCNET-537 - WebDashboard is not able to handle project names with special characters (like #)
  • CCNET-538 - Executing Null Task should produce a successful build
  • CCNET-540 - If source control labelling fails the build results will not get published

  • CCNET-545 - CCNet should strip illegal characters out of external content before merging it into the build log
  • CCNET-546 - EmailPublisher should send an email even if there is an exception while building the message
  • CCNET-547 - VSS WorkingDirectory not working correctly
  • CCNET-557 - MSBuild task buildArgs configuration property gives problems when adding more switches

New Feature

  • CCNET-512 - Use a Server/Project tree in CCTray instead of a dropdown list – better usability for lots of projects.
  • CCNET-530 - Allow users to set timeout in seconds, minutes, hours etc.
  • CCNET-533 - Allow user to specify a custom notification message on success/failure
  • CCNET-558 - CCTray should give an estimate of build time remaining

Improvement

  • CCNET-511 - Vault task seems to always get the "full" tree – makes getting source slow and leads to large _sgbak directories

  • CCNET-525 - CCTray should show last build time
  • CCNET-534 - Subversion executable should be option

  • CCNET-544 - Allow location of dashboard.config to be itself configurable
  • CCNET-548 - CCTray: allow user to specify custom icons
  • CCNET-549 - Change modification date format so that dates are more easily sortable
  • CCNET-550 - Change dateformat for dashboard to ISO 8601
  • CCNET-554 - Subversion should rebase its working directory from the project working directory
  • CCNET-555 - TrunkUrl, WorkingDirectory and Executable should be optional parameters for Subversion
